  ## Blog Writing Rule

  - git commit 後に `BLOG_REMINDER` hook が発火したら、`/tech-blog` スキルを実行する
    （ブログ化の価値判断はスキル内で行う）
  - ユーザーが「ブログにして」「知見を残して」等と言った場合も `/tech-blog` スキルを実行する
    （何を書くか聞き返さず、スキル内で会話分析・候補提案を行う）
  - **AI試行錯誤トリガー**: 以下のいずれかを検知し、最終的に動作する状態に到達したら、
    その場で即座にブログ化を提案する（コミットまで待たない）:
    - AIが最初に選んだアプローチでエラーが発生し、別のアプローチで解決した
    - ユーザーの指示が曖昧でAIが誤解し、指示を具体化・変更して解決した
    - 同じ目的でやり直し・方針転換が発生した
    - 記事の切り口: ❌ 最初の指示/アプローチ → ✅ 改善後の指示/アプローチ の対比を提示する

  つまり、/tech-blog が起動するタイミングは3種類

  ┌──────────────┬─────────────────────────────────────────────────────────────────────────┐
  │   トリガー   │                                起動契機                                 │
  ├──────────────┼─────────────────────────────────────────────────────────────────────────┤
  │ gitコミット  │ BLOG_REMINDER hook が発火したとき                                       │
  │ 後           │                                                                         │
  ├──────────────┼─────────────────────────────────────────────────────────────────────────┤
  │ ユーザー明示 │ 「ブログにして」「知見を残して」等の発話                                │
  │ 指示         │                                                                         │
  ├──────────────┼─────────────────────────────────────────────────────────────────────────┤
  │ AI試行錯誤検 │ アプローチ変更・指示具体化・やり直しが発生し、最終的に動作した瞬間（コ  │
  │ 出           │ ミット待たず即提案）                                                    │
  └──────────────┴─────────────────────────────────────────────────────────────────────────┘

      }
    ]
  }

  スクリプト: C:\Users\masat\.claude\hooks\blog-reminder.sh（11行）

  #!/bin/bash
  INPUT=$(cat)
  if echo "$INPUT" | grep -q '"command".*git commit'; then
    echo "BLOG_REMINDER: This commit may contain blog-worthy technical insights. Evaluate whether to
  invoke /tech-blog skill."
  fi
  exit 0

  流れまとめ

  git commit を Bash 実行
    ↓
  PostToolUse hook が blog-reminder.sh を実行
    ↓
  コマンドに "git commit" が含まれていれば
    ↓
  "BLOG_REMINDER: ..." メッセージを出力
    ↓
  私（Claude Code）がそれを受けて、CLAUDE.md の Blog Writing Rule に従い /tech-blog 起動を判断

  つまり、CLAUDE.mdの「ルール」と settings.jsonの「フック」と shell script
  の「検出ロジック」の3点セットで動いています。すべて実在しており、整合しています。
