# AI秘書による開発・組織改善 — 5W1H

## Why（なぜ）

- バイブコーディングはしているが、やり方が人それぞれで属人化している
- 人によってアウトプットの品質に差がある
- 開発ノウハウが共有されず、同じ失敗を繰り返している
- GitHubは完成コードの置き場で、開発プロセスには使っていない

## What（何を）

- 全員にAI秘書をつけ、秘書どうしが知識を共有する仕組みをつくる
- 開発の進め方（壁打ち→設計→実装）を統一する
- 組織の働き方（会議・ナレッジ共有・引き継ぎ）も改善する

## Who（誰が）

- DXソリューショングループ全員
- 開発者だけでなく、プログラムしないメンバーも対象

## When（いつ）

| Step | 期間 | 内容 |
|------|------|------|
| Step 0 | 1-2ヶ月目 | 土台をつくる（GitHub活用開始・秘書セットアップ・共通ルール） |
| Step 1 | 3-4ヶ月目 | 開発を変える（壁打ちフロー・失敗共有・テックブログ） |
| Step 2 | 5-6ヶ月目 | 知識をつなげる（3層構造・レビュー蓄積・タスク連動） |
| Step 3 | 7-8ヶ月目 | 組織に広げる（議事録自動化・ナレッジ蓄積・新メンバー支援） |

## Where（どこで）

- GitHub：開発プロセス（Issue・PR・レビュー）の基盤
- AI秘書（Claude Code）：個人の窓口＋チーム知識の共有基盤
- 課内で展開（部署横断ではなく、まず課の中で完結）

## How（どうやって）

23の施策を段階的に実施する。

### 開発編（18項目）

| # | 施策 |
|---|------|
| 1 | 「こうしたい」をIssueに書く |
| 2 | 秘書と壁打ちしてIssueの中身を詰める |
| 3 | 秘書が設計書を自動で作る |
| 4 | 秘書がテストとコードを自動で書く |
| 5 | 秘書がPR（変更提案）を自動で出す |
| 6 | チームで見て、採用するか決める |
| 7 | タスク管理アプリとIssueを連動させる |
| 8 | メンバー全員にAI秘書をつける |
| 9 | チーム共通のルール（CLAUDE.md）を共有する |
| 10 | 秘書どうしで知識を共有する（3層構造） |
| 11 | 毎朝秘書が「こうしたらどう？」と提案 |
| 12 | 秘書に話しかけるだけで全部始まる |
| 13 | ラベルを1つ貼るだけで自動修正+PR |
| 14 | 危険な操作は自動でブロック（安全装置） |
| 15 | レビュー結果を秘書に覚えさせる |
| 16 | 過去の壁打ち内容を自動で参照する |
| 17 | 失敗事例をチームで共有する |
| 18 | テックブログで開発ノウハウを共有 |

### 組織編（5項目）

| # | 施策 |
|---|------|
| 19 | 会議の議事録を自動で記録・共有 |
| 20 | 決定事項をタスクに自動反映 |
| 21 | 秘書に聞けば進捗・経緯がすぐ分かる |
| 22 | 新メンバーの秘書が自動でキャッチアップ |
| 23 | 課内ナレッジの蓄積・検索 |
