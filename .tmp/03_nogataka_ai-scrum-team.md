- **プラットフォーム**: Qiita

## 概要

Claude Codeのagents、skills、hooksを活用し、8つの専門AIエージェントで構成される仮想スクラムチームを構築。個人開発者でもチームレベルのコード品質を維持できるテンプレート「AI Sprint Team」を紹介。

## 8つの専門AIエージェント

| エージェント | 役割 |
|------------|------|
| product-owner | 優先順位付けと受け入れ基準の定義 |
| planner | 要件分析とタスク分解 |
| backend-dev | データベースとAPI実装 |
| frontend-dev | UIコンポーネント開発 |
| reviewer | コードレビューとセキュリティ監査 |
| tester | テスト設計と実行 |
| scrum-master | 進捗管理と通知 |
| reporter | ドキュメントと報告 |

## ワークフロー（5フェーズ）

1. **ドメイン知識の記述**（初回のみ）
2. **GitHub Issueの作成**（基本的な説明を記入）
3. **`/sprint-plan 1 "goal"`** でスプリント計画を自動生成
4. **`/implement [issue-number]`** で設計→実装→レビュー→テスト→PR作成を自動実行
5. **`/retro`** でレトロスペクティブ実施

## 技術的特徴

### GitHub Issues統合

- `gh` CLIでバックログ管理
- "ready"ラベルでスプリント対象を管理

### 自動セーフティガード（6つのhooks）

- 再帰的削除の防止
- force pushの防止
- 環境変数の漏洩防止
- 権限エスカレーションの防止
- （他2つ）

### 品質自動化

- TypeScriptファイル変更時にESLint + Prettierを自動実行
- reviewerエージェントがセキュリティ問題を人間レビュー前に検出

## 実践例

Next.js 15 + TypeScript + Supabaseでプロジェクト管理アプリを構築:

`/implement 1` コマンド1つで以下を自動実行:
1. RLSポリシー付きスキーマ設計
2. APIルート実装
3. Reactコンポーネント作成
4. 自動テスト
5. PR生成

reviewerがSupabase認証の脆弱性（anonキーの不正使用）を検出し、自動的にbackend-devに修正を差し戻した事例あり。

## 重要なポイント

- **効率**: 人間の作業はIssue作成とコマンド実行のみ
- **品質管理**: 専門エージェントが一貫したコード標準・セキュリティ・テストカバレッジを維持
- **学習統合**: レトロスペクティブごとにプロジェクトルールを更新、手動ドキュメント更新不要で継続改善
- **カスタマイズ**: `.claude/rules/`ファイルとドメイン知識ドキュメントでエージェント動作を適応
