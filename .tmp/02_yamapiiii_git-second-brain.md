# 思考をすべてGitに書き出し、Claude Codeに運用させている話

- **公開日**: 2026-03-01
- **プラットフォーム**: Zenn

## 概要

Gitリポジトリを「セカンドブレイン」として運用し、Claude Codeに整理・探索・実行を委任するシステム。単なるナレッジマネジメントではなく、日常の知的作業をGitとAIに乗せる実験。

## 1日の運用フロー

| 時間帯 | 操作 | コマンド |
|--------|------|---------|
| 朝 | 未処理Inbox・放置PJ確認 | `/brain-status` |
| 移動中 | 音声入力で思考をInbox投入 | - |
| 仕事中 | 並列調査実行 | `/deep-research` |
| 企画時 | 過去の関連情報を自動探索 | `/kickoff` |
| タスク実行 | GitHub Issueを渡して自動実装 | `/work` |
| アイデア出し | 無限生成（寝る前に起動） | `/idea --infinite` |
| 週末 | 自動振り返り | `/weekly-review` |

## フォルダ構造

番号で情報の流れ（入力→熟成→知識化→卒業）を表現:

```
brain/
├── 00_Inbox/           # 入力
├── 10_Journal/         # 熟成
├── 20_Projects/        # 熟成
├── 30_Tech_Notes/      # 知識化
├── 50_Business_Context/# 知識化
├── 90_System/          # システム
└── 99_Archives/        # 卒業
```

## CLAUDE.mdの3つの重要ルール

1. **「思考パートナー」宣言**: AIを単なるコーディング補助ではなく、思考構造化と問いかけの相手として位置づけ
2. **確認なしで動作**: 「承認を求めずに即実行すること。迷ったら実行してから報告」で流れを阻害しない
3. **偽情報禁止**: brain内に記録されていない情報は「未記録」と明示するルール

## カスタムコマンド（14種、主要なもの）

| コマンド | 機能 | 使用頻度 |
|---------|------|--------|
| `/work` | Issue自動実行 | 毎日 |
| `/kickoff` | 関連知識並列探索 | よく使う |
| `/deep-research` | Web検索+内部探索 | 週2-3回 |
| `/idea` | アイデア生成 | 週1-2回 |
| `/weekly-review` | 自動振り返り | 毎週 |
| `/brain-status` | 朝の確認 | 毎日 |
| `/voice-note` | 音声入力の整理 | 毎日 |
| `/orchestrate` | 自律運用 | 必要時 |

「work/kickoff/voice-noteの3つで日常の8割をカバー」

## brain/project分離戦略

- **brain**: Why/What（意思決定ログ、思考の経緯）
- **project**: How（技術的実装）
- 完全に分離することで、仕様変更の経緯が失われない設計

## PM Layer（自動介入型プロジェクト管理）

- 新規プロジェクト作成時に`pm_brief.md`が自動生成
- 全プロジェクト横断のヘルス診断（キャパ超過、散漫度、競合検出）
- 放置検出と停滞パターン分析

ヘルス状態の可視化:
- 🟢 健全（brief存在・指標定義済・直近活動あり）
- 🟡 注意（2週間以上放置）
- 🔴 対処（30日以上放置）

## `/orchestrate`（自律運用機能）

- AIが優先度スコアリングに基づいて自動判断・実行
- 5軸評価: 依存関係・ラベル・タイプ・工数・鮮度
- ガードレール: 1チェーン最大3Issue、10分/Issue

## Gitを選んだ理由

- Claude Codeがファイルシステムを直接操作 → APIやプラグイン不要
- GitHub Actionsで自動化容易
- gitログによるバージョン管理
- フォルダ構造の自由度

## 適性

- **向いている**: CLI習熟者、テキスト思考習慣者、Claude Code日常利用者、PM/PdM
- **向かない**: GUIユーザー、セットアップ回避志向者

## スタートアップ

最小構成3ステップ:
1. フォルダ作成（00_Inbox、10_Journal等）
2. CLAUDE.mdの作成
3. Inboxへの記録開始

「2週間、書き出すだけ試してほしい。コマンドは後から追加可能」
