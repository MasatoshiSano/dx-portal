## このファイルの位置づけ
チームの秘書（Claude Code）が常に読む基本ルール。
気づいたらその場で追記 PR を出す。200 行を超えたら刈り込み。

## 開発の流れ
1. **15 分以上かかる作業は Issue を立てる**（それ未満は直接 PR で OK）
2. **壁打ち**で方針を決めたら、4 項目テンプレで Issue にコメント投稿
3. **Draft PR を早めに出す**（途中確認用）
4. 完了したら `/review-my-changes` でセルフレビュー
5. 人間レビュー → マージ
6. 学びがあれば CLAUDE.md 追記 PR を出す

## 秘書への基本指示
- **過去の失敗事例を先に検索してから**実装に入る
- 分からないことは勝手に決めず、**選択肢を提示して人間に聞く**
- **テストは実装前に書く**（TDD）
- `any` / `unknown` は使わない
- 判断に迷ったら、**プロジェクト側か個人メモ**に寄せる（共通に上げるのは合意後）

## 壁打ちログのテンプレ（Issue にコメント投稿する形式）
壁打ちの結論を残すときは、この 4 項目で：

```
【前提】
（どんな状況・背景で議論したか）

【検討した選択肢】
- 案A: ...
- 案B: ...
- 案C: ...

【結論】
（採用した案）

【却下理由】
（他の案をなぜ選ばなかったか）
```
